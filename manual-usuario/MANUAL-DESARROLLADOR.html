1. Introduccion

CKJ - Sistema de Almacen es una aplicacion web SPA para gestion de inventarios con arquitectura cliente-servidor. El frontend Angular 22 se comunica con una API REST Node.js/Express que persiste los datos en SQLite.

Flujo de Autenticacion

CLIENTE                              SERVIDOR
  │                                      │
  │  POST /api/login                     │
  │  {username, password}                │
  │─────────────────────────────────→    │
  │                    Busca usuario en BD│
  │                    bcrypt.compare()   │
  │                    jwt.sign({id,rol}) │
  │  ←─────────────────────────────────  │
  │  {token, user}                       │
  │                                      │
  │  [Guarda token en localStorage]      │
  │                                      │
  │  GET /materiales                     │
  │  Authorization: Bearer <token>      │
  │─────────────────────────────────→    │
  │                    jwt.verify(token)  │
  │                    queryAll(SQL)      │
  │  ←─────────────────────────────────  │
  │  [{materiales...}]                   │
  │                                      │
  │  [Token expira en 12h → redirect login]
El token JWT contiene: {id, username, rol}. Se almacena en localStorage bajo la clave ckj_token. El usuario se guarda en ckj_user.

4. Base de Datos (SQLite)

La BD se maneja con sql.js (SQLite compilado a WebAssembly). Se almacena en un archivo local backend/database.sqlite que se crea automaticamente al iniciar el servidor.

La BD se genera automaticamente con datos de prueba si no existe. Para resetear: borra backend/database.sqlite y reinicia el servidor.

Esquema Completo

CREATE TABLE materiales (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre          TEXT NOT NULL,
    descripcion     TEXT DEFAULT '',
    categoria       TEXT DEFAULT '',
    cantidad        REAL NOT NULL DEFAULT 0,
    unidad          TEXT DEFAULT 'unidades',
    ubicacion       TEXT DEFAULT '',
    fechaIngreso    TEXT DEFAULT (date('now')),
    proveedor       TEXT DEFAULT '',
    precioUnitario  REAL DEFAULT 0,
    stockMinimo     REAL DEFAULT 10
);

CREATE TABLE movimientos (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    materialId      INTEGER NOT NULL,
    tipo            TEXT NOT NULL CHECK(tipo IN ('ingreso','salida')),
    cantidad        REAL NOT NULL,
    fecha           TEXT DEFAULT (datetime('now','localtime')),
    proveedor       TEXT DEFAULT '',
    destino         TEXT DEFAULT '',
    usuario         TEXT DEFAULT 'Admin',
    comprobante     TEXT DEFAULT '',
    FOREIGN KEY (materialId) REFERENCES materiales(id)
);

CREATE TABLE usuarios (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    username        TEXT UNIQUE NOT NULL,
    password        TEXT NOT NULL,
    nombre          TEXT NOT NULL,
    rol             TEXT NOT NULL DEFAULT 'almacen',
    activo          INTEGER DEFAULT 1
);

Relaciones

TablaRelacionTipo

Seed Data

Sistema de Rutas

Definido en app.routes.ts. Usa lazy loading con loadComponent() y proteccion con AuthGuard:

export const routes: Routes = [
  { path: '', redirectTo: '/dashboard', pathMatch: 'full' },

  // Sin auth (publica)
  { path: 'login', loadComponent: () => import('./componentes/login/login').then(m => m.LoginComponent) },

  // Con auth (cualquier rol)
  { path: 'dashboard',  loadComponent: ..., canActivate: [AuthGuard] },
  { path: 'materiales', loadComponent: ..., canActivate: [AuthGuard] },

  // Con auth + roles especificos
  { path: 'ingreso', loadComponent: ..., canActivate: [AuthGuard], data: { roles: ['admin', 'almacen'] }},
  { path: 'salida',  loadComponent: ..., canActivate: [AuthGuard], data: { roles: ['admin', 'almacen', 'ventas'] }},
  { path: 'ventas',  loadComponent: ..., canActivate: [AuthGuard], data: { roles: ['admin', 'ventas'] }},
  { path: 'reportes', loadComponent: ..., canActivate: [AuthGuard], data: { roles: ['admin'] }},
  { path: 'usuarios', loadComponent: ..., canActivate: [AuthGuard], data: { roles: ['admin'] }},

  { path: '**', redirectTo: 'dashboard' }
];

Flujo de Autenticacion Frontend

1. Login: AuthService.login() → POST /api/login → recibe {token, user}
   → localStorage.setItem('ckj_token', token)
   → localStorage.setItem('ckj_user', JSON.stringify(user))
   → userSubject.next(user)
   → router.navigate(['/dashboard'])

2. Cada peticion HTTP: AuthInterceptor.intercept()
   → Toma ckj_token del localStorage
   → Si existe, agrega header: Authorization: Bearer <token>

3. Proteccion de rutas: AuthGuard.canActivate()
   → isLoggedIn()? Si no → redirect /login
   → route.data.roles existe? hasRole()? Si no → redirect /dashboard

4. Logout: AuthService.logout()
   → removeItem('ckj_token'), removeItem('ckj_user')
   → userSubject.next(null)
   → router.navigate(['/login'])

12. Guia: Agregar una Nueva Feature

Ejemplo completo: Agregar una nueva pagina "Proveedores" al sistema.

Paso 1: Crear el componente Angular

ng generate component componentes/proveedores

Esto crea: proveedores.ts, proveedores.html, proveedores.css, proveedores.spec.ts

Paso 2: Agregar la ruta

En app.routes.ts:

{
  path: 'proveedores',
  loadComponent: () => import('./componentes/proveedores/proveedores').then(m => m.ProveedoresComponent),
  canActivate: [AuthGuard],
  data: { roles: ['admin'] }
}

Paso 3: Agregar endpoint en el backend

En backend/server.js:

// Crear tabla (en initDatabase)
db.run(`CREATE TABLE IF NOT EXISTS proveedores (...)`);

// Crear rutas
app.get('/api/proveedores', authMiddleware(), (req, res) => {
  res.json(queryAll('SELECT * FROM proveedores'));
});

app.post('/api/proveedores', authMiddleware(['admin']), (req, res) => {
  const { nombre, telefono } = req.body;
  runSql('INSERT INTO proveedores VALUES (NULL,?,?)', [nombre, telefono]);
  res.status(201).json(queryOne('SELECT * FROM proveedores WHERE id=?', [getLastId()]));
});

13. Troubleshooting

Error: Token expirado o invalido

Causa: El token JWT expiro (12h) o el servidor se reinicio con diferente JWT_SECRET.

Solucion: Hacer logout y login nuevamente. Si persiste, verifica que JWT_SECRET sea consistente.

Error: database.sqlite no encontrada / corrupta

Causa: La BD se corrompio o fue borrada.

Solucion: Detener el servidor, borrar backend/database.sqlite, reiniciar. Se recreara automaticamente con seed data.

Error: CORS bloquea peticiones

Causa: El frontend (puerto 4200) intenta llamar al backend (puerto 3000) y CORS no esta configurado.

Solucion: Verificar que app.use(cors()) este en server.js.

Error: npm install falla con husky

Causa: Husky intenta instalar hooks de git y el directorio .git no existe o hay permisos.

Solucion: git init primero, luego npm install. O temporalmente: npm install --ignore-scripts.

Error: No se ven los cambios en Angular

Causa: Cache del navegador o compilacion atascada.

Solucion: Ctrl + Shift + R (recarga forzada) o ng serve --force.

Error: Husky bloquea commit

Causa: Gitleaks detecto un posible secreto o Prettier fallo.

Solucion: Corrige el secreto o usa git commit --no-verify solo si es falso positivo. Verifica con npx husky run pre-commit.

Error: Puerto 3000 o 4200 ya en uso

Causa: Otro proceso ocupando el puerto.

Solucion: netstat -ano | findstr :3000 para identificar el PID, luego taskkill /PID <PID> /F.
